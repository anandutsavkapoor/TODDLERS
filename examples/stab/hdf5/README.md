# recollapse data (regenerated, NOT shipped)

The recollapse HDF5 (`recollapse_data_<MODEL_PREFIX>.hdf5`) is **derived from the
evolution runs** and is regenerated by `run_interpolant_generator.py`
(`save_recollapse_data`). It is intentionally NOT shipped: a static copy would be
silently reused against different/modified evolution runs and give wrong SFR-normed
SEDs.

If you change or replace the underlying evolution runs, delete any existing
`recollapse_data_*.hdf5`/`.h5` here so it is regenerated fresh. The campaign scripts
force this regeneration.
